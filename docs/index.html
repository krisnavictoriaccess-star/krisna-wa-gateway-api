Krisna WA Gateway API

Referensi Lengkap Integrasi Gateway Enterprise

🌍 Base URL

https://api.krisnadev.my.id

PERINGATAN KEAMANAN: SIMPAN API KEY ANDA!

Sistem ini menggunakan arsitektur keamanan tingkat perbankan (One-Way SHA-256 Hash). Artinya, saat API Key dibuat, server TIDAK PERNAH menyimpan API Key asli Anda. Server hanya menyimpan versi acak (Hashed) yang tidak bisa dikembalikan lagi.

Oleh karena itu, Anda WAJIB menyimpan API Key (Plain Text) Anda di tempat yang aman setelah di-generate. Jika Anda menghilangkannya, Admin pun tidak akan bisa melihat atau memulihkannya kembali. Anda harus menghapus key lama dan membuat key yang baru.

🔐 Autentikasi (Headers)

Selalu sertakan kredensial di header HTTP (bukan di URL parameter).

HeaderStatusDeskripsi
x-master-key Wajib (Admin) Untuk akses endpoint master.
x-api-key Wajib (Klien) Token klien yang didapat dari Generate.
sender_id Opsional Pilih nomor pengirim spesifik. Kosong = Rotator.

⚡ WebSockets

API ini menyediakan WebSocket untuk memantau status sesi perangkat (Koneksi WA) secara *real-time*.

SOCKET.IO https://api.krisnadev.my.id

Contoh Implementasi Klien (HTML/JavaScript):

<!-- Load library Socket.IO Client -->
<script src="https://cdn.socket.io/4.7.5/socket.io.min.js"></script>
<script>
  // Inisialisasi koneksi Socket.IO
  const socket = io("https://api.krisnadev.my.id", {
    transports: ["websocket", "polling"]
  });

  // Mendengarkan event 'device_status'
  socket.on("device_status", (data) => {
    console.log("Nomor:", data.device, "| Status:", data.status);
    
    if (data.status === "WAITING_PAIRING") {
       console.log("Masukkan kode pairing ini di WhatsApp Anda:", data.code);
    }
    else if (data.status === "CONNECTED") {
       console.log("WhatsApp berhasil terhubung dan siap digunakan!");
    }
  });
</script>

POST /package/add
Menambahkan paket layanan baru ke dalam sistem.
Requires x-master-key
Parameters
FieldTypeStatusDeskripsi
x-master-key String (Header) wajib Kunci rahasia master server.
nama_paket String wajib Nama unik untuk paket (contoh: 'Premium', 'VIP').
deskripsi String opsional Deskripsi fitur paket.
harga String opsional Harga paket (contoh: '50000').
limit_pesan Integer opsional Batas pesan per bulan. Default: 1000.
limit_device Integer opsional Jumlah WhatsApp yang bisa dikoneksikan. Default: 1.
limit_autoreply Integer opsional Batas jumlah auto-reply. Default: 5.
fitur_broadcast Boolean opsional Aktifkan fitur kirim pesan massal? Default: false.
fitur_media Boolean opsional Aktifkan kirim gambar/video/dokumen? Default: false.
fitur_group Boolean opsional Aktifkan kirim pesan ke grup? Default: false.
fitur_webhook Boolean opsional Aktifkan Webhook URL? Default: false.
fitur_vcard Boolean opsional Aktifkan kirim kontak (vCard)? Default: false.
fitur_lokasi Boolean opsional Aktifkan kirim koordinat lokasi? Default: false.
fitur_polling Boolean opsional Aktifkan pembuatan polling (vote)? Default: false.
fitur_contact_list Boolean opsional Aktifkan tarik kontak perangkat? Default: false.
fitur_group_list Boolean opsional Aktifkan tarik grup perangkat? Default: false.
fitur_inbox Boolean opsional Aktifkan penyimpanan riwayat pesan masuk? Default: false.
is_public Boolean opsional true = Bisa dibeli umum. false = Paket rahasia/internal. Default: true.
Contoh Request Body
{
  "nama_paket": "Premium VIP",
  "deskripsi": "Paket tanpa batas untuk pengusaha",
  "harga": "150000",

  "limit_device": 5,
  "limit_autoreply": 100,
  "fitur_broadcast": true,
  "fitur_media": true,
  "fitur_group": true,
  "fitur_webhook": true,
  "fitur_vcard": true,
  "fitur_lokasi": true,
  "fitur_polling": true,
  "fitur_contact_list": true,
  "fitur_group_list": true,
  "fitur_inbox": true,
  "is_public": true
}
Contoh Response
Berhasil (200 / 201)
{
  "status": true,
  "message": "Paket berhasil dibuat",
  "data": { "id": 2, "nama_paket": "Premium VIP", "..." : "..." }
}
Gagal (4xx / 5xx)
{ "status": false, "message": "nama_paket wajib diisi." }
