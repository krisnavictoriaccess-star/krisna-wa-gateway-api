Krisna WA Gateway API

Referensi Lengkap Integrasi Gateway Enterprise

🌍 Base URL

https://api.krisnamarket.my.id

🔐 Autentikasi (Headers)

Selalu sertakan kredensial di header HTTP (bukan di URL parameter).

HeaderStatusDeskripsi
x-master-key Wajib (Admin) Untuk akses endpoint master (manajemen klien).
x-api-key Wajib (Klien) Token klien yang didapatkan dari proses Generate.
sender_id Opsional Digunakan pada fitur Messaging untuk memilih nomor pengirim spesifik. Kosongkan untuk mode Rotator.

👑 Admin / Master (Manajemen API Key)

GET /api-key/list
Melihat daftar semua akun klien (API Key) yang terdaftar di database.
Requires x-master-key
Contoh Response
Berhasil (200 / 201)
{
  "status": true,
  "data": [
    {
      "key": "KEY-ABC123DEF456",
      "label": "Toko Utama",
      "paket": "Premium",

      "terpakai_bulan_ini": 1500,
      "sisa_kuota": 48500,
      "status": "active",
      "expired_at": "2026-12-31T23:59:59Z"

  ]

{
  "status": false,
  "message": "Akses ditolak. Master Key salah atau tidak ditemukan."

POST /auto-reply/add
Membuat robot balasan otomatis berdasarkan kata kunci spesifik.
Requires x-api-key
Parameters
Field Type Status Deskripsi
sender_id String (Header) wajib Nomor device/bot yang akan merespon diletakkan di HTTP Header.
keyword String wajib Kata pemicu.
response String wajib Balasan yang akan dikirim bot.
match_type String opsional exact (sama persis) / contains (mengandung). Default: exact.
media_url String opsional Tautan URL publik untuk mengirim gambar/video/dokumen bersamaan dengan respon.
media_type String opsional Tipe file media (contoh: image, video, document).
Contoh Request Body (JSON)
{
  "keyword": "harga",
  "response": "Berikut adalah daftar harga kami:",
  "match_type": "exact",
  "media_url": "https://domain.com/brosur.jpg",
  "media_type": "image"
}
Contoh Response
Berhasil (200 / 201)
{
  "status": true,
  "message": "Auto-Reply berhasil ditambahkan."
}
Gagal (4xx / 5xx)
{
  "status": false,
  "message": "Keyword atau response tidak boleh kosong."
}
